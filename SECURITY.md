# Security Policy

A safety/guardrail tool. It evaluates a policy and returns a decision; it does not itself kill
processes. A defect here is safety-relevant — treat a rule that fails *open* (lets a run continue
when it should halt) as security-grade.

Report issues **in this tool** privately: GitHub **Report a vulnerability** on this repo, or email
the contact on the org profile. Target: **5 business days** to acknowledge, coordinated disclosure.
